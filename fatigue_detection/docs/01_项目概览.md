# 项目概览

## 1. 项目目标

这是一个运行在本地设备上的轻量化疲劳检测原型，目标不是训练一个大型视频模型，而是完成一条边缘智能闭环：

`视频输入 -> 人脸关键点 -> 疲劳特征 -> 轻量模型 -> ONNX Runtime -> 本地实时告警`

项目重点有 4 个：

- 本地实时运行，不依赖云端推理
- 用可解释特征体现疲劳检测思路
- 完成从训练到部署的完整链路

## 2. 当前实现到了什么程度

当前项目已经具备：

- 摄像头或视频文件实时推理
- MediaPipe 人脸关键点检测
- `EAR`、`MAR`、头姿 `pitch/yaw/roll` 特征提取
- 滑动窗口统计特征构建
- 轻量 `MLP` 训练与 `ONNX` 导出
- `ONNX Runtime` 本地推理
- 状态平滑、连续告警、冷却控制
- 中英文切换界面与简洁演示模式
- 规则基线、推理性能对比、特征消融实验

## 3. 为什么这里既有“规则”又有“模型”

这套系统不是直接把原始视频送进神经网络，而是先从视频里提取结构化疲劳特征，再做分类。

具体分成两层：

- 规则层：例如低 `EAR`、高 `MAR`、连续闭眼、连续哈欠等，可以直接构成启发式判断
- 模型层：将这些特征组合成一个更灵活的分类边界，减少单阈值规则的脆弱性

所以现在项目里同时保留：

- 一个规则基线，用来做实验对比
- 一个轻量 MLP，用来体现 AI 训练与部署闭环

## 4. 数据集和标签怎么来

当前主要使用 `UTA-RLDD`，位置在：

`data/raw_videos/UTA-RLDD/`

目录按受试者划分，每个受试者下面通常有 `0`、`5`、`10` 这类视频文件。当前项目主要使用 `0` 和 `10` 做二分类子集。

标签经历了两步演进：

1. 视频级标签  
   `0 -> Normal`，`10 -> Fatigue Risk`

2. 窗口级标签  
   通过 `data/labels/window_labels_review.csv` 只保留更可信的疲劳风险区间，减少整段视频继承标签带来的噪声

## 5. 当前最关键的实验结论

当前基于 `6` 个受试者构造的精化数据集：

- 数据集：`data/dataset_cv6_review.npz`
- 样本数：`10915`
- 类别分布：`normal = 7046`，`fatigue = 3869`

目前最重要的结论有 3 个：

1. 窗口级标签后，分组交叉验证结果明显优于之前的粗标签版本。
2. 在这个子集上，规则基线依然很强，说明标签质量和特征选择比模型复杂度更重要。
3. `eye + mouth` 的窗口统计特征比“全特征直接全部输入”更稳。
