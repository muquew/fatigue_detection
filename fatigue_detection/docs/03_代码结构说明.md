# 代码结构说明

## 1. 总体结构

项目代码主要分成两部分：

- `src/`：在线检测与部署
- `train/`：离线训练与实验

理解项目时，建议不要先从训练脚本看起，而是先看在线入口，再回头看数据和训练。

## 2. src 目录说明

### 2.1 `src/main.py`

这是整个项目的主入口，负责串起在线链路：

1. 打开摄像头或视频
2. 调用人脸关键点检测
3. 提取当前帧疲劳特征
4. 推入滑动窗口
5. 生成窗口特征向量
6. 调用 ONNX 推理
7. 进行投票平滑
8. 触发连续告警
9. 显示 UI

### 2.2 `src/config.py`

集中管理默认配置，例如：

- 采集分辨率
- 预览窗口尺寸
- `window_size`
- `EAR/MAR` 阈值
- 投票窗口大小
- 告警阈值和冷却时间
- 默认语言
- 模型和结果文件路径

真实演示中的大多数微调都在这里完成。

### 2.3 `src/camera.py`

负责读取摄像头或视频文件，并向主程序提供帧数据包。

### 2.4 `src/landmarker.py`

负责加载 MediaPipe 的 `face_landmarker.task` 并输出关键点结果。

这里还处理了一个实际问题：

- 如果模型路径带中文，MediaPipe 可能打不开
- 代码会把模型复制到临时 ASCII 路径再加载

### 2.5 `src/features.py`

负责从关键点计算每帧特征，当前核心是：

- `EAR`
- `MAR`
- 头姿 `pitch/yaw/roll`

这是从“关键点”到“可训练特征”的桥梁。

### 2.6 `src/window_buffer.py`

负责把单帧特征组合成时间窗口特征。

当前窗口特征包括：

- 当前帧的 `ear_left`、`ear_right`、`ear_avg`、`mar`、`pitch`、`yaw`、`roll`
- `ear_mean`
- `ear_min`
- `mar_mean`
- `mar_max`
- `low_ear_streak`
- `high_mar_streak`
- `close_eye_ratio`
- `yawn_ratio`

### 2.7 `src/infer_onnx.py`

负责加载：

- `mlp.onnx`
- `scaler.pkl`

然后完成：

- 输入标准化
- ONNX Runtime 推理
- 概率后处理
- 疲劳概率阈值过滤

如果模型不存在，它会回退到启发式规则。

### 2.8 `src/alert.py`

负责告警门控逻辑。当前不是“判断一次疲劳就报警”，而是同时要求：

- 平滑状态为 `Fatigue Risk`
- 当前置信度达到阈值
- 当前投票比例达到阈值
- 连续满足若干次
- 两次报警之间超过冷却时间

### 2.9 `src/ui.py`

负责界面叠加显示。当前功能包括：

- 默认精简信息面板
- 详情模式
- 中英文切换
- 中文字体渲染
- 告警横幅显示

### 2.10 `src/classifier.py`

定义轻量 MLP 结构，以及 checkpoint 的保存和加载方法。

## 3. train 目录说明

### 3.1 `train/extract_features.py`

把原始视频逐帧处理成特征 CSV。

输入：

- 视频文件或视频目录

输出：

- 每个视频一个 CSV

### 3.2 `train/build_dataset.py`

把逐帧特征 CSV 转成窗口级训练数据集。

输入：

- 特征 CSV
- 可选的窗口标签 CSV

输出：

- `.npz` 训练数据集

### 3.3 `train/generate_window_label_review.py`

根据启发式规则生成一份“待复核/可直接使用”的窗口标签区间文件，用来减少人工标注工作量。

### 3.4 `train/train_mlp.py`

训练轻量 MLP，并做按受试者分组的交叉验证。

它当前的重点不是追求训练集分数，而是评估：

- 没见过的受试者上能不能泛化

### 3.5 `train/export_onnx.py`

把 PyTorch 模型导出成 ONNX。

### 3.6 `train/evaluate.py`

对保存的模型做离线评估。

### 3.7 `train/rule_baseline.py`

用纯规则法在同一份数据集上做基线对比。

### 3.8 `train/benchmark_inference.py`

比较：

- PyTorch 前向
- ONNX 前向
- ONNX 端到端预测

### 3.9 `train/feature_ablation.py`

做特征组合实验，帮助判断哪些特征真正有效。

### 3.10 `train/summarize_results.py`

把实验结果汇总成便于阅读的 Markdown。

## 4. 推荐阅读顺序

如果你要快速理解代码，建议顺序如下：

1. `src/main.py`
2. `src/config.py`
3. `src/features.py`
4. `src/window_buffer.py`
5. `src/infer_onnx.py`
6. `src/alert.py`
7. `src/ui.py`
8. `train/build_dataset.py`
9. `train/train_mlp.py`

## 5. 当前代码里最值得注意的事实

1. 在线检测和离线训练使用的是同一套特征定义，不是两套逻辑。
2. 模型输入不是原始图像，而是 15 维窗口统计特征。
3. 当前误触发控制主要依赖概率阈值、投票平滑和连续告警门控。
4. 当前真实瓶颈是标签质量和跨受试者泛化，不是推理速度。
