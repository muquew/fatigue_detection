# 使用说明

## 1. 环境准备

你当前使用的环境是：

```bash
conda activate fatigue
```

安装依赖：

```bash
pip install -r requirements.txt
```

关键文件：

- `models/face_landmarker.task`
- `models/mlp.pth`
- `models/scaler.pkl`
- `models/mlp.onnx`

如果模型文件缺失，也可以先跑 `dry-run` 验证入口。

## 2. 最常用的运行方式

### 2.1 快速自检

```bash
python src/main.py --dry-run --frames 20
```

用途：

- 检查主程序入口是否正常
- 检查 ONNX 推理链路是否可加载
- 检查状态平滑和告警逻辑是否正常

### 2.2 直接开摄像头

```bash
python src/main.py
```

用途：

- 做实时演示
- 观察界面叠加
- 测试闭眼、哈欠、低头时的反应

### 2.3 用本地视频文件演示

```bash
python src/main.py --source path/to/demo.mp4
```

如果是 `mov` 文件也支持，例如：

```bash
python src/main.py --source path/to/demo.mov
```

## 3. 运行时按键

- `L`：切换中英文界面
- `I`：展开或收起详细信息
- `Q` / `ESC`：退出

默认界面只显示关键信息，适合录屏演示；展开详情后可查看原始状态、投票比例、疲劳概率、EAR、MAR 和头姿。

## 4. 一条完整训练闭环怎么跑

### 4.1 从原始视频提取逐帧特征

```bash
python train/extract_features.py --input data/raw_videos/UTA-RLDD --output-dir data/features_cv6 --frame-stride 5
```

输出是每个视频一个 `csv`，每一行代表一个被保留下来的帧样本。

### 4.2 生成窗口标签文件

```bash
python train/generate_window_label_review.py \
  --input-dir data/features_cv6 \
  --output data/labels/window_labels_review.csv
```

这是当前项目里的“半自动精标”步骤，用来为疲劳视频筛选更可信的窗口区间。

### 4.3 生成训练数据集

```bash
python train/build_dataset.py \
  --input-dir data/features_cv6 \
  --window-labels data/labels/window_labels_review.csv \
  --output data/dataset_cv6_review.npz
```

### 4.4 训练模型

```bash
python train/train_mlp.py \
  --dataset data/dataset_cv6_review.npz \
  --epochs 8 \
  --batch-size 64 \
  --lr 0.001 \
  --seed 42
```

### 4.5 导出 ONNX

```bash
python train/export_onnx.py
```

### 4.6 评估与实验

```bash
python train/evaluate.py --dataset data/dataset_cv6_review.npz
python train/rule_baseline.py --dataset data/dataset_cv6_review.npz
python train/benchmark_inference.py --dataset data/dataset_cv6_review.npz --num-samples 256
python train/feature_ablation.py --dataset data/dataset_cv6_review.npz --epochs 5 --batch-size 64 --lr 0.001 --seed 42
python train/summarize_results.py
```

1. `python src/main.py --dry-run --frames 20`
